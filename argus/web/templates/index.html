Welcome to the Bitcoin Argus testnet playground

A friendly little machine that exists to make Bitcoin development easier. It runs a whole menu of Bitcoin test networks — with nodes, indexers, explorers and payment tooling — so you can build, break and learn without spending a single real satoshi. Pick a network, point your tools at it, and start hacking. 🛰️

Which test network should I use?

Reach for the most contained option that still exercises what you need, then move outward only if you must: regtest → custom signet → mutinynet → signet → testnet4 → testnet3.

💡 For your own local development and testing, prefer Bitcoin's regtest whenever possible — run it on your own machine and you control mining, coins, and resets entirely. You can set it up with a single command, and try more advanced techniques like double-spends with this tutorial. The networks below are a shared playground for when you need something others can reach too.

Who can access what

This is a shared, hosted playground. Some things are open to you, the visitor; others are controlled by the site operator and are not reachable over the internet.

You (a visitor) can

  • Point an Electrum wallet at the public Fulcrum servers
  • Use the Cashu mints, browse the mempool explorers, and use Bitcart
  • Open Lightning connections to the public LND P2P ports

Only the operator can

  • Reach Bitcoin Core's RPC and LND gRPC/REST/admin (bound to the server's localhost)
  • Produce blocks — e.g. regtest blocks are mined by the operator (~1/min), not by you
  • Reset, reconfigure, or restart any service

In the tables below, a public port is reachable by you; an operator port is localhost-only on the server.
